DATABASE SCHEMA
================================================================================

USERS
  - id (primary key)
  - username (unique)
  - email (unique)
  - password_hash
  - full_name
  - created_at

CONTESTS
  - id (primary key)
  - creator_id (foreign key)
  - title
  - description
  - start_time, end_time
  - created_at

PROBLEMS
  - id (primary key)
  - contest_id (foreign key)
  - title
  - description
  - difficulty
  - time_limit, memory_limit
  - created_at

SUBMISSIONS
  - id (primary key)
  - user_id, problem_id (foreign keys)
  - code
  - language
  - status
  - score
  - submitted_at

CONTEST_REGISTRATIONS
  - user_id, contest_id (unique pair)
  - registered_at

API ENDPOINTS
================================================================================

AUTHENTICATION
  POST   /register           - Register new user
  POST   /login              - User login
  GET    /logout             - User logout

CONTESTS
  GET    /contests/          - List all contests
  GET    /contests/<id>      - View specific contest
  POST   /contests/create    - Create new contest
  POST   /contests/<id>/register - Register for contest

PROBLEMS
  GET    /problems/<id>      - View problem details

SUBMISSIONS
  POST   /submissions/submit - Submit code
  GET    /submissions/<id>   - View submission
  GET    /submissions/my     - User's submissions

LEADERBOARD
  GET    /leaderboard/       - Global leaderboard
  GET    /leaderboard/contest/<id> - Contest leaderboard

GETTING STARTED
================================================================================

1. INSTALL DEPENDENCIES
   chmod +x start.sh
   ./start.sh

   This will:
   - Create Python virtual environment
   - Install Flask, MySQL connector, Werkzeug
   - Initialize database
   - Start the server

2. OPEN THE APP
   Visit: http://localhost:5000

3. CREATE ACCOUNT
   - Click "Sign Up"
   - Fill in details
   - Create account

4. CREATE CONTEST
   - Go to Contests
   - Click "Create Contest"
   - Fill contest details
   - Submit

5. ADD PROBLEMS
   - View your contest
   - Add problems with descriptions and constraints

6. PARTICIPATE
   - Register for contests
   - Solve problems
   - Submit solutions
   - Check leaderboard

TROUBLESHOOTING
================================================================================

If you encounter issues:

1. MySQL Connection Error
   → Start MySQL: sudo systemctl start mysql
   → Check credentials in .env file

2. Port 5000 in Use
   → Kill process: lsof -ti:5000 | xargs kill -9
   → Or change port in run.py

3. Python Module Errors
   → Activate venv: source venv/bin/activate
   → Reinstall: pip install -r requirements.txt

4. Page Not Found (404)
   → Make sure you're logged in
   → Check URL is correct

See SETUP.md and QUICKSTART.md for more troubleshooting tips.
